如何使用Vibe Coding开发全栈应用？

发表于 2026-02-19 分类于 blog 本文字数： 1.3k 阅读时长 ≈ 1 分钟

听了一位使用Vibe Coding的前辈的演讲，他分享了他使用Vibe Coding的一些思路和方法，遂写下来记录一下。

他在自己创业的过程中，大量应用了Vibe Coding进行前后端全栈应用开发。我认为他的一些方法和观点简单有效、不无道理。

LLM的特性及其解决方案

  • 对话控制在10轮以内，多了以后LLM可能会丢失记忆
  • 严格使用git进行版本控制，不要依赖AI的代码回退功能。原因是对于AI做出的修改，我们是不可控的
  • 如果多轮对话（2-3轮）后，LLM结果结果还是不行，果断回退代码（git reset --hard或者git revert），不要为丢失代码感到惋惜。开启新会话后，AI工具很快就能追回原来的进度
  • 上述方法要是还不行，果断更换模型尝试，即使是稍差一点的模型都没事
  • 让AI工具（例如Claude Code）同时生成测试代码，并编写钩子（Hook），生成完毕后自动执行所有测试
  • 重视回归测试，这也对应了上一个观点中，需要运行所有测试的内容。原因还是一样，对于AI的修改，我们是不可控制的（保不准它就偷偷修改了之前已经测试过的代码 ( ﹁ ﹁ ) ~→）
  • 尽量少修改CLAUDE.md/AGENT.md等文件，LLM生成的内容大概率已经很不错了

使用Vibe Coding工具进行全栈开发

  • 使用Plan模式，先讨论出项目需求方案，确定项目所需的技术栈
  • 确定好后，让LLM形成文档，保存下来
  • 使用Build模式，快速制作出前端的demo
    • 使用HTML输出，因为它是最通用的前端描述方法，适配各种框架
    • 一图胜千言，必要的时候可以画出草稿上传给LLM，让它构造相应的HTML
    • 对于需要向后端请求的数据展示部分，先使用Mock Data
    • 在HTML生成完毕后，让LLM根据HTML蓝图，使用相关技术栈构造出前端网站及Mock Data
  • 前端做好后，基于前端进行后端的搭建
    • 利用前端构建出的Mock Data，搭建数据库
    • 利用数据库结构和Mock Data的数据，搭建出后端对应的Schema，以及相关接口，并形成接口文档
  • 后端搭建完成并测试后，进行前后端数据对接工作
    • 让LLM根据接口文档，将原有的Mock Data修改为从后端获取实际数据的逻辑

使用Vibe Coding工具进行文档写作

我现在主要还是使用Opencode作为主要的LLM工具，用它上面的免费模型 （收费的模型对于学生党来说还是有点贵了） ，搭配相应的skills来辅助我写文档（报告 / ppt），下面是我进行文档写作时常用的方法和skills。

  • 我先大致编辑好文档的内容，放在项目的目录中
  • 在目录当中创建好.opencode/skills文件夹（或者.claude/skills文件夹）
  • 按需求放入下面的skills
    • anthropics docx、xlsx、pptx、pdf：用于读取我写好的文档，或者按照文档的要求输出为对应的格式
    • editor：让LLM基于我编辑好的内容，调整成比较正式的格式和语言
    • humanizer-zh：增强文档的人味 （能增强一点是一点了）
  • 先规划，告诉LLM具体的使用需求后，再让它进行写作
  • 对于文档写作类任务，我感觉不太需要使用/init让LLM生成记忆文档，毕竟不是什么复杂的任务，执行init还挺费时间的
